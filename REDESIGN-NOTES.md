## 2. 예약 흐름 (기존 시스템, 그대로 활용)

백엔드: **sinbiweb PHP 솔루션** — 라우터 `home.php?go=...`, 처리 `core/res/rev.act.php`.

```
① 날짜/룸 선택 → ② 예약정보 입력 → ③ 결제하기 → ④ 예약완료
  rev.make        rev.make.input    (결제)       (완료)
```

| 단계 | 페이지(go=) | 내용 |
|---|---|---|
| ① 날짜/룸 선택 | `rev.make` | 예약날짜(달력) + 지점 선택(드롭다운, **6개점**) → 테마별 시간 슬롯. **예약가능(클릭O)/예약마감(클릭X)**, 일부 슬롯 **할인 배지**. 가능 슬롯 클릭 → ② |
| ② 예약정보 입력 | `rev.make.input` | 지점·일시·테마 확인 + **예약자명 / 연락처(010+2칸) / 인원**(선택 시 가격 자동계산) + 할인정보 + **결제방식: 예약금 20,000원** + **이용약관 동의(라디오)** → **「예약하기」**(`fun_submit()`) → `core/res/rev.act.php` POST |
| ③ 결제하기 | (제출 후) | 예약금 20,000원 결제. ※ 입력 페이지엔 **온라인 결제모듈(PG) 미탑재** — 제출 후 결제 단계에서 호출되는 구조로 추정 (실제 제출 없이는 미확인) |
| ④ 예약완료 | — | 예약번호 발급 |

**부가 흐름**
- **예약확인** (`rev.login`): **예약자명 + 연락처 + 예약번호**로 조회
- **취소/변경 정책**: 시작 24시간 이내 취소·변경 시 예약금 환불 불가 · 10분 전 미도착 시 게임시간 차감
